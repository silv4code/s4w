# Security Policy

S4W is a passive web security assessment toolkit intended for owned assets, controlled labs and formally authorized environments.

The project is designed to support defensive security reviews, hardening checks, exposure analysis and structured reporting for web applications.

---

## Supported Usage

S4W should only be used in the following scenarios:

- assets you own;
- controlled lab environments;
- internal security reviews;
- authorized penetration testing engagements;
- developer security checks;
- defensive hardening assessments.

Do not use S4W against third-party systems without explicit permission.

---

## Scope of the Tool

S4W focuses on passive and low-impact checks, including:

- security header review;
- passive recon;
- visible endpoint analysis;
- cookie flag inspection;
- CSP and CORS review;
- passive JavaScript indicators;
- hardening gaps;
- metadata exposure;
- structured vulnerability indicators.

S4W does not perform brute force, automatic exploitation, destructive testing, aggressive fuzzing, credential attacks, evasion or persistence.

---

## Reporting Security Issues

To report a security issue related to this project, use one of the following channels:

- open a private security advisory through the GitHub repository;
- contact the repository maintainer through the official GitHub profile;
- create an issue only if the report does not expose sensitive details.

Do not publish exploit details, sensitive findings or working attack chains in public issues.

---

## Responsible Disclosure

When reporting a vulnerability in S4W itself, include:

- affected version;
- operating system;
- Python version;
- steps to reproduce;
- expected behavior;
- actual behavior;
- potential impact;
- suggested remediation, if available.
